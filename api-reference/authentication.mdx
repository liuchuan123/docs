Every request to the PayMatrix API must be authenticated. PayMatrix uses Bearer token authentication — you include your secret API key in the `Authorization` header of each request. There are no cookies, sessions, or OAuth flows required. Keep reading to learn how to obtain your keys, which key to use in each environment, and how to keep them secure.

## How to Authenticate

Pass your API key as a Bearer token in the `Authorization` header on every request:

```bash
curl https://api.paymatrix.com/v1/payments \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```

<Note>
  Requests that omit the `Authorization` header entirely, or that supply a malformed value, will receive a `401 Unauthorized` response immediately — no partial processing occurs.
</Note>

## API Key Formats

PayMatrix issues two types of API keys, each with a distinct prefix so you can immediately tell them apart:

| Key type    | Prefix       | Environment               |
| ----------- | ------------ | ------------------------- |
| Test key    | `pm_test_`   | Sandbox (no real charges) |
| Live key    | `pm_live_`   | Production (real charges) |

Use **test keys** while building and testing your integration. Payments, orders, and refunds created with a test key are simulated and never result in real financial transactions. Switch to a **live key** only when you are ready to accept real payments.

## Where to Find Your Keys

You can view and manage your API keys in the PayMatrix Dashboard:

1. Log in to the [PayMatrix Dashboard](https://dashboard.paymatrix.com).
2. Navigate to **Settings → API Keys**.
3. Copy the key for the environment you need.

<Tip>
  If you suspect a key has been compromised, rotate it immediately from the same **Settings → API Keys** page. PayMatrix will issue a new key and invalidate the old one without disrupting existing settled transactions.
</Tip>

## Unauthenticated Response

If you make a request without a valid API key, the API returns a `401 Unauthorized` response with the following JSON body:

```json
{
  "error": {
    "code": "authentication_required",
    "message": "No valid API key provided.",
    "request_id": "req_XXXXXXXX"
  }
}
```

Save the `request_id` value if you need to contact PayMatrix support — it uniquely identifies the failed request in the system logs.

## Security Best Practices

Following these practices keeps your API keys — and your users' financial data — safe:

<Warning>
  **Never expose your live API key in client-side code, public repositories, or logs.** Live keys authorize real financial transactions. Anyone who obtains your live key can create charges, trigger refunds, and access sensitive merchant data on your behalf.
</Warning>

- **Use environment variables.** Store keys in environment variables (e.g., `PAYMATRIX_API_KEY`) and read them at runtime. Never hardcode a key directly into your source code.
- **Never commit keys to version control.** Add your `.env` file to `.gitignore` and audit your repository history if you accidentally push a key.
- **Rotate keys regularly.** Treat API keys like passwords and rotate them on a schedule, or immediately after any suspected exposure.
- **Use test keys in non-production environments.** Your CI pipeline, staging servers, and local development should always use `pm_test_` keys.
- **Apply the principle of least privilege.** If PayMatrix introduces scoped API keys in the future, grant each key only the permissions it needs.

<CodeGroup>

```bash Environment variable (shell)
export PAYMATRIX_API_KEY="pm_live_xxxxxxxxxxxx"

curl https://api.paymatrix.com/v1/payments \
  -H "Authorization: Bearer $PAYMATRIX_API_KEY"
```

```python Environment variable (Python)
import os
import requests

api_key = os.environ["PAYMATRIX_API_KEY"]

response = requests.get(
    "https://api.paymatrix.com/v1/payments",
    headers={"Authorization": f"Bearer {api_key}"}
)
```

```javascript Environment variable (Node.js)
const apiKey = process.env.PAYMATRIX_API_KEY;

const response = await fetch("https://api.paymatrix.com/v1/payments", {
  headers: { Authorization: `Bearer ${apiKey}` },
});
```
