Your merchant profile contains the core account information associated with your PayMatrix integration. The profile is scoped to the API key you use in the request, so there are no path or query parameters — you always receive the profile for the authenticated merchant. Use this endpoint to verify account status, confirm your settlement currency, or surface business details in your internal dashboards.

## Endpoint

```
GET https://api.paymatrix.com/v1/merchants/profile
```

## Request Example

<CodeGroup>
```bash cURL
curl https://api.paymatrix.com/v1/merchants/profile \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```
</CodeGroup>

## Response — 200 OK

```json
{
  "id": "mch_AABBCCDD",
  "business_name": "Acme Corp",
  "email": "billing@acme.com",
  "status": "active",
  "settlement_currency": "USD",
  "country": "US",
  "created_at": "2023-06-01T00:00:00Z"
}
```

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for your merchant account, prefixed with `mch_`. This ID is stable and will not change for the lifetime of your account.
</ResponseField>

<ResponseField name="business_name" type="string">
  The legal business name registered on your PayMatrix account, as provided during onboarding.
</ResponseField>

<ResponseField name="email" type="string">
  The primary billing email address associated with your merchant account. Financial notices and settlement reports are sent to this address.
</ResponseField>

<ResponseField name="status" type="string">
  The current operational status of your merchant account. See [Account Statuses](#account-statuses) below for details on each value.
</ResponseField>

<ResponseField name="settlement_currency" type="string">
  The ISO 4217 currency code used for all settlement payouts to your bank account (e.g., `USD`). This is configured during onboarding and can only be changed by contacting PayMatrix support.
</ResponseField>

<ResponseField name="country" type="string">
  The two-letter [ISO 3166-1 alpha-2](https://www.iso.org/iso-3166-country-codes.html) country code of your registered business address (e.g., `US`, `GB`, `DE`).
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when your merchant account was created.
</ResponseField>

## Account Statuses

| Status | Description |
|---|---|
| `pending` | Your account has been created but identity verification or onboarding steps are still incomplete. Live payment processing is not yet available. |
| `active` | Your account is fully verified and in good standing. You can process live payments and receive settlements. |
| `suspended` | Your account has been temporarily suspended. Payment processing is disabled. Contact PayMatrix support to resolve the suspension. |

<Note>
  If your account status is `pending`, you can still use test-mode API keys (prefixed with `pm_test_`) to integrate and validate your implementation before going live.
</Note>
