---
title: "GET /merchants/settlements — List Settlements"
sidebarTitle: "Settlements"
description: "List settlement batches for your merchant account. Filter by date range and status. Returns gross, fee, and net amounts per settlement period."
---

PayMatrix batches your processed payments into daily settlement cycles and transfers the net amount to your bank account after deducting processing fees. This endpoint gives you a paginated list of all settlement records for your merchant account, along with a breakdown of gross revenue, fees charged, and the net amount transferred. Use the date range and status filters to narrow results for reconciliation or reporting workflows.

## Endpoint

```
GET https://api.paymatrix.com/v1/merchants/settlements
```

## Query Parameters

<ParamField query="from" type="string">
  Filter settlements with a `period_start` on or after this date. Must be a valid ISO 8601 date or datetime string (e.g., `2024-01-01` or `2024-01-01T00:00:00Z`).
</ParamField>

<ParamField query="to" type="string">
  Filter settlements with a `period_end` on or before this date. Must be a valid ISO 8601 date or datetime string. When used with `from`, this defines the inclusive date range.
</ParamField>

<ParamField query="status" type="string">
  Filter by settlement status. Accepted values: `pending`, `processing`, `completed`, `failed`. Omit this parameter to return settlements of all statuses.
</ParamField>

<ParamField query="page" type="integer" default="1">
  The page number to retrieve. Use in combination with `per_page` to paginate through large result sets. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of settlement records to return per page. Defaults to `20`. Maximum value is `100`.
</ParamField>

## Request Example

<CodeGroup>
```bash cURL
curl "https://api.paymatrix.com/v1/merchants/settlements?from=2024-01-01&status=completed" \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```
</CodeGroup>

### Settlement Object Fields

<ResponseField name="id" type="string">
  Unique identifier for the settlement batch, prefixed with `set_`.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the settlement. See [Settlement Statuses](#settlement-statuses) below.
</ResponseField>

<ResponseField name="gross_amount" type="integer">
  The total value of all payments included in this settlement, in the smallest currency unit (e.g., cents for USD), before fees are deducted.
</ResponseField>

<ResponseField name="fee_amount" type="integer">
  The total PayMatrix processing fees deducted from the gross amount for this settlement period, in the smallest currency unit.
</ResponseField>

<ResponseField name="net_amount" type="integer">
  The amount actually transferred to your bank account (`gross_amount` minus `fee_amount`), in the smallest currency unit.
</ResponseField>

<ResponseField name="currency" type="string">
  ISO 4217 currency code for all amounts in this settlement record.
</ResponseField>

<ResponseField name="payment_count" type="integer">
  The number of individual payment transactions included in this settlement batch.
</ResponseField>

<ResponseField name="period_start" type="string">
  ISO 8601 timestamp marking the start of the settlement period (typically 00:00:00 UTC).
</ResponseField>

<ResponseField name="period_end" type="string">
  ISO 8601 timestamp marking the end of the settlement period (typically 23:59:59 UTC).
</ResponseField>

<ResponseField name="settled_at" type="string">
  ISO 8601 timestamp of when the settlement funds were actually transferred to your bank account. This field is `null` for settlements that have not yet reached `completed` status.
</ResponseField>

## Settlement Statuses

| Status | Description |
|---|---|
| `pending` | Payments are being collected into this settlement batch. The period has not yet closed. |
| `processing` | The settlement period has closed and the payout transfer has been initiated. |
| `completed` | Funds have been successfully transferred to your bank account. |
| `failed` | The payout transfer failed. PayMatrix will retry automatically or contact you if manual action is needed. |

<Tip>
  Subscribe to the `settlement.completed` webhook event to receive instant notification when a settlement is paid out, rather than polling this endpoint.
</Tip>
