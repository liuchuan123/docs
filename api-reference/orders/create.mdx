---
title: "POST /orders — Create an Order"
sidebarTitle: "Create Order"
description: "Create a payment intent order in PayMatrix. Specify amount and currency, then attach a payment to capture funds. Returns the Order object."
---

When you create an order, PayMatrix generates a payment intent that holds the specified amount in the requested currency. Orders begin in a `pending` state and remain open until a payment is attached, the order expires, or you explicitly cancel it. You can optionally attach metadata and a human-readable description to keep your internal records aligned with PayMatrix transactions.

## Endpoint

```
POST https://api.paymatrix.com/v1/orders
```

## Request Body

<ParamField body="amount" type="integer" required>
  The order amount in the **smallest currency unit** (e.g., cents for USD). For example, `5000` represents $50.00 USD. Must be a positive integer greater than zero.
</ParamField>

<ParamField body="currency" type="string" required>
  Three-letter [ISO 4217](https://www.iso.org/iso-4217-currency-codes.html) currency code in uppercase (e.g., `USD`, `EUR`, `GBP`). Must be a currency supported by your merchant account.
</ParamField>

<ParamField body="description" type="string">
  A human-readable label for this order. Useful for reconciliation and appears in your PayMatrix dashboard. Maximum 255 characters.
</ParamField>

<ParamField body="metadata" type="object">
  A set of custom key-value pairs you can attach to the order. Keys and values must be strings. Metadata is returned on all future retrievals of this order and is never used by PayMatrix for processing logic.

  <Expandable title="metadata constraints">
    - Maximum **20 keys** per object
    - Key length: up to 40 characters
    - Value length: up to 500 characters
  </Expandable>
</ParamField>

<ParamField body="expires_in" type="integer" default="3600">
  Number of seconds from order creation until the order automatically transitions to `expired`. Defaults to `3600` (1 hour). Set a higher value for orders that may take longer to complete checkout.
</ParamField>

## Request Example

<CodeGroup>
```bash cURL
curl -X POST https://api.paymatrix.com/v1/orders \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000,
    "currency": "USD",
    "description": "Order #1001 - Pro Plan",
    "metadata": {
      "customer_id": "cust_abc123",
      "internal_ref": "INV-2024-001"
    }
  }'
```
</CodeGroup>

## Response — 201 Created

A successful request returns the newly created Order object with a `201 Created` status code.

```json
{
  "id": "ord_1A2B3C4D5E",
  "status": "pending",
  "amount": 5000,
  "currency": "USD",
  "description": "Order #1001 - Pro Plan",
  "payment_id": null,
  "metadata": {
    "customer_id": "cust_abc123",
    "internal_ref": "INV-2024-001"
  },
  "created_at": "2024-01-15T10:30:00Z",
  "expires_at": "2024-01-15T11:30:00Z"
}
```

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the order, prefixed with `ord_`. Use this ID to retrieve, update, or associate a payment with this order.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status of the order. Newly created orders always start as `pending`. Possible values: `pending`, `paid`, `expired`, `cancelled`.
</ResponseField>

<ResponseField name="amount" type="integer">
  The order amount in the smallest currency unit, exactly as provided in the request.
</ResponseField>

<ResponseField name="currency" type="string">
  The ISO 4217 currency code for this order, exactly as provided in the request.
</ResponseField>

<ResponseField name="description" type="string">
  The human-readable description attached to this order, or `null` if none was provided.
</ResponseField>

<ResponseField name="payment_id" type="string">
  The ID of the payment associated with this order, or `null` if no payment has been attached yet. This field is populated once a payment attempt is made against the order.
</ResponseField>

<ResponseField name="metadata" type="object">
  The custom key-value metadata attached to this order, or an empty object if none was provided.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the order was created.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 timestamp of when the order will expire if unpaid. Derived from `created_at` plus the `expires_in` value supplied at creation.
</ResponseField>

<Note>
  Store the `id` returned in the response — you'll need it to attach a payment and track the order's lifecycle. Orders cannot be recreated; expired orders must be replaced with a new order.
</Note>
