title: "GET /orders/{id} — Retrieve an Order"
sidebarTitle: "Retrieve Order"

---

Retrieving an order gives you a real-time snapshot of its current state, including whether a payment has been attached and what lifecycle status the order is in. Use this endpoint to poll for status updates, verify amounts before displaying a receipt, or look up metadata you originally attached at order creation.

## Endpoint

```
GET https://api.paymatrix.com/v1/orders/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the order to retrieve. Order IDs are prefixed with `ord_` (e.g., `ord_1A2B3C4D5E`).
</ParamField>

## Request Example

<CodeGroup>
```bash cURL
curl https://api.paymatrix.com/v1/orders/ord_1A2B3C4D5E \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```
</CodeGroup>

## Response — 200 OK

```json
{
  "id": "ord_1A2B3C4D5E",
  "status": "paid",
  "amount": 5000,
  "currency": "USD",
  "description": "Order #1001 - Pro Plan",
  "payment_id": "pay_9Z8Y7X6W5V",
  "metadata": { "customer_id": "cust_abc123" },
  "created_at": "2024-01-15T10:30:00Z",
  "expires_at": "2024-01-15T11:30:00Z"
}
```

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the order, prefixed with `ord_`.
</ResponseField>

<ResponseField name="status" type="string">
  The current lifecycle status of the order. See the [Order Statuses](#order-statuses) section below for descriptions of each value.
</ResponseField>

<ResponseField name="amount" type="integer">
  The order amount in the smallest currency unit (e.g., cents for USD). An amount of `5000` with currency `USD` represents $50.00.
</ResponseField>

<ResponseField name="currency" type="string">
  ISO 4217 three-letter currency code for this order (e.g., `USD`, `EUR`, `GBP`).
</ResponseField>

<ResponseField name="description" type="string">
  Human-readable description attached to this order, or `null` if none was set.
</ResponseField>

<ResponseField name="payment_id" type="string">
  The ID of the payment associated with this order. This is `null` for `pending` orders and is populated once a payment attempt is made. Use this ID to retrieve full payment details from the Payments API.
</ResponseField>

<ResponseField name="metadata" type="object">
  Custom key-value pairs attached to the order at creation or via a subsequent update.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the order was originally created.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 timestamp of when the order will expire or did expire. For `paid` or `cancelled` orders, this timestamp may be in the past.
</ResponseField>

## Order Statuses

Understanding the status field helps you build reliable order lifecycle logic in your integration.

| Status | Description |
|---|---|
| `pending` | The order has been created and is awaiting payment. It can still be updated or cancelled. |
| `paid` | A payment has been successfully captured against this order. No further changes are possible. |
| `expired` | The order passed its `expires_at` timestamp without a successful payment. |
| `cancelled` | The order was explicitly cancelled before payment. |

<Tip>
  Rather than polling this endpoint on a timer, consider subscribing to the `order.expired` and `payment.succeeded` webhook events to receive real-time status updates with no additional API calls.
</Tip>
