The PayMatrix REST API gives you programmatic access to payments, orders, merchants, settlements, and webhooks. All communication happens over HTTPS using JSON — every request you send and every response you receive will use `application/json`. The sections below cover the conventions that apply across every endpoint in this reference.

## Base URL

All API requests are made to the following base URL. The version identifier (`v1`) is part of the path and must be included in every request.

```
https://api.paymatrix.com/v1
```

<Note>
  PayMatrix uses URL-based versioning. When a new major version is released, the path segment will change (e.g., `/v2`). You will always be notified before a version is deprecated.
</Note>

## Request Format

How you pass data to the API depends on the HTTP method you are using:

- **POST / PATCH** — Send parameters as a JSON body with the `Content-Type: application/json` header.
- **GET** — Pass parameters as URL query strings (e.g., `?page=2&per_page=50`).

```bash
# POST example — JSON body
curl https://api.paymatrix.com/v1/orders \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"amount": 4999, "currency": "usd"}'

# GET example — query parameters
curl "https://api.paymatrix.com/v1/payments?page=1&per_page=20" \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```

## Response Format

Every response from the PayMatrix API is a JSON object.

- **Single resources** — Fields are returned at the root level of the response object.
- **List responses** — Results are returned inside a `data` array alongside a `pagination` object.

```json
// Single resource
{
  "id": "pay_AbCdEfGh",
  "amount": 4999,
  "currency": "usd",
  "status": "succeeded"
}

// List response
{
  "data": [
    { "id": "pay_AbCdEfGh", "amount": 4999, "currency": "usd", "status": "succeeded" },
    { "id": "pay_IjKlMnOp", "amount": 1200, "currency": "usd", "status": "pending" }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 243,
    "total_pages": 13
  }
}
```

See [Pagination](/api-reference/pagination) for full details on navigating list responses.

## Dates and Times

All timestamps in the API use **ISO 8601** format in UTC. When you supply a date parameter or read a date field in a response, expect this format:

```
2024-01-15T10:30:00Z
```

## Amounts and Currencies

Monetary amounts are always expressed as **integers in the smallest currency unit** to avoid floating-point precision issues.

| Currency | Amount value | Represents |
| -------- | ------------ | ---------- |
| USD      | `100`        | $1.00      |
| USD      | `4999`       | $49.99     |
| JPY      | `500`        | ¥500       |

<Warning>
  Zero-decimal currencies such as JPY do not use a subunit. Passing `500` for JPY means ¥500, not ¥5.00. Always confirm the decimal rules for your target currency before processing.
</Warning>

## Idempotency

To safely retry POST requests without accidentally creating duplicate resources, include an `Idempotency-Key` header. PayMatrix stores the result of the first request with a given key and returns that same result for any subsequent retries within **24 hours**.

```bash
curl https://api.paymatrix.com/v1/payments \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: a8f3b2c1-4e67-4d89-b012-3f5a6c7d8e9f" \
  -d '{"amount": 4999, "currency": "usd", "order_id": "ord_XxYyZz"}'
```

<Tip>
  Use a randomly generated UUID v4 as your idempotency key. Reuse the same key only when retrying the exact same logical operation.
</Tip>

## Authentication

Requests are authenticated using a Bearer token in the `Authorization` header. See the [Authentication](/api-reference/authentication) page for key formats, sandbox vs. production keys, and security best practices.

## Errors

When something goes wrong, PayMatrix returns a structured JSON error object with a machine-readable `code`, a human-readable `message`, and a `request_id` you can share with support. See the [Errors](/api-reference/errors) page for the full list of error codes and HTTP status meanings.

## Endpoint Quick Reference

The table below lists every available endpoint in the PayMatrix API.

| Method  | Endpoint                      | Description              |
| ------- | ----------------------------- | ------------------------ |
| `POST`  | `/orders`                     | Create an order          |
| `GET`   | `/orders/{id}`                | Retrieve an order        |
| `PATCH` | `/orders/{id}`                | Update an order          |
| `POST`  | `/payments`                   | Create a payment         |
| `GET`   | `/payments/{id}`              | Retrieve a payment       |
| `GET`   | `/payments`                   | List payments            |
| `POST`  | `/payments/{id}/refund`       | Refund a payment         |
| `GET`   | `/merchants/profile`          | Get merchant profile     |
| `GET`   | `/merchants/settlements`      | List settlements         |
| `POST`  | `/webhooks`                   | Register a webhook       |
| `GET`   | `/webhooks/events`            | List webhook events      |
