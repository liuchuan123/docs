---
title: "POST /payments — Create a Payment"
sidebarTitle: "Create Payment"
description: "Create a payment by attaching a payment method to an existing order. Accepts card, bank transfer, or digital wallet. Returns the Payment object."
---

To create a payment, you submit a payment method alongside an existing order ID. PayMatrix will attempt to authorize and capture the payment immediately, returning a Payment object that reflects the outcome. You can attach card details, bank transfer credentials, or a digital wallet token depending on your integration. For safe retries without double-charging, include an `Idempotency-Key` header with every request.

<Note>
  Card numbers are never stored on your servers. All sensitive card data is encrypted in transit and handled in PayMatrix's PCI-DSS compliant environment.
</Note>

## Endpoint

```
POST https://api.paymatrix.com/v1/payments
```

## Request Headers

| Header | Description |
|---|---|
| `Authorization` | **Required.** Bearer token — e.g. `Bearer pm_test_xxxxxxxxxxxx` |
| `Content-Type` | **Required.** Must be `application/json` |
| `Idempotency-Key` | **Recommended.** Unique string to safely retry requests |

<Tip>
  Generate your `Idempotency-Key` from a UUID or a hash of the order ID and timestamp. The same key replays the original response for up to 24 hours without creating a duplicate payment.
</Tip>

## Request Body

<ParamField body="order_id" type="string" required>
  The ID of the existing order you want to pay. The order must be in a `pending` state and belong to your merchant account.
</ParamField>

<ParamField body="method" type="string" required>
  The payment method type. Accepted values: `card`, `bank_transfer`, `wallet`.
</ParamField>

<ParamField body="card" type="object">
  Card details. Required when `method` is `card`.

  <Expandable title="card fields">
    <ParamField body="number" type="string">
      The full card number as a string with no spaces or separators — e.g. `"4242424242424242"`.
    </ParamField>

    <ParamField body="exp_month" type="integer">
      The card's expiry month as an integer between `1` and `12`.
    </ParamField>

    <ParamField body="exp_year" type="integer">
      The card's expiry year as a 4-digit integer — e.g. `2027`.
    </ParamField>

    <ParamField body="cvc" type="string">
      The card security code. Either 3 digits (Visa, Mastercard) or 4 digits (Amex).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="bank_transfer" type="object">
  Bank transfer details. Required when `method` is `bank_transfer`.

  <Expandable title="bank_transfer fields">
    <ParamField body="account_number" type="string">
      The bank account number to debit.
    </ParamField>

    <ParamField body="routing_number" type="string">
      The 9-digit ABA routing number associated with the bank account.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="metadata" type="object">
  An optional set of key-value pairs you can attach to the payment for your own record-keeping. Keys and values must be strings. Up to 20 keys are permitted, with values up to 500 characters each.
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.paymatrix.com/v1/payments \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: unique-key-abc123" \
  -d '{
    "order_id": "ord_1A2B3C4D5E",
    "method": "card",
    "card": {
      "number": "4242424242424242",
      "exp_month": 12,
      "exp_year": 2027,
      "cvc": "123"
    }
  }'
```

```python Python
import paymatrix

client = paymatrix.Client(api_key="pm_test_xxxxxxxxxxxx")

payment = client.payments.create(
    order_id="ord_1A2B3C4D5E",
    method="card",
    card={
        "number": "4242424242424242",
        "exp_month": 12,
        "exp_year": 2027,
        "cvc": "123",
    },
    idempotency_key="unique-key-abc123",
)
```

```javascript Node.js
import PayMatrix from "paymatrix";

const client = new PayMatrix("pm_test_xxxxxxxxxxxx");

const payment = await client.payments.create(
  {
    order_id: "ord_1A2B3C4D5E",
    method: "card",
    card: {
      number: "4242424242424242",
      exp_month: 12,
      exp_year: 2027,
      cvc: "123",
    },
  },
  { idempotencyKey: "unique-key-abc123" }
);
```

</CodeGroup>

## Response — 201 Created

A successful request returns a `201 Created` status and a Payment object in the response body.

```json Example Response
{
  "id": "pay_9Z8Y7X6W5V",
  "order_id": "ord_1A2B3C4D5E",
  "status": "succeeded",
  "amount": 5000,
  "currency": "USD",
  "method": "card",
  "card": {
    "brand": "visa",
    "last4": "4242",
    "exp_month": 12,
    "exp_year": 2027
  },
  "metadata": {},
  "created_at": "2024-01-15T10:30:05Z",
  "settled_at": null
}
```

## Response Fields

<ResponseField name="id" type="string">
  The unique identifier for this payment, prefixed with `pay_`.
</ResponseField>

<ResponseField name="order_id" type="string">
  The ID of the order this payment is associated with.
</ResponseField>

<ResponseField name="status" type="string">
  The current payment status. On creation this will be `succeeded`, `pending`, or `failed` depending on the outcome of the authorization attempt.
</ResponseField>

<ResponseField name="amount" type="integer">
  The payment amount in the smallest unit of the currency — e.g. cents for USD. A value of `5000` represents $50.00.
</ResponseField>

<ResponseField name="currency" type="string">
  The three-letter ISO 4217 currency code — e.g. `USD`.
</ResponseField>

<ResponseField name="method" type="string">
  The payment method used: `card`, `bank_transfer`, or `wallet`.
</ResponseField>

<ResponseField name="card" type="object">
  Present when `method` is `card`. Contains a safe, tokenized view of the card — the full number is never returned.

  <Expandable title="card fields">
    <ResponseField name="brand" type="string">
      The card network — e.g. `visa`, `mastercard`, `amex`.
    </ResponseField>

    <ResponseField name="last4" type="string">
      The last four digits of the card number.
    </ResponseField>

    <ResponseField name="exp_month" type="integer">
      The card's expiry month.
    </ResponseField>

    <ResponseField name="exp_year" type="integer">
      The card's expiry year.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="metadata" type="object">
  The key-value metadata you attached to the request, or an empty object if none was provided.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the payment was created.
</ResponseField>

<ResponseField name="settled_at" type="string">
  ISO 8601 timestamp of when the payment funds were settled to your account. Returns `null` until settlement is complete.
</ResponseField>
