---
title: "GET /payments — List Payments"
sidebarTitle: "List Payments"
description: "List all payments for your merchant account. Filter by status, date range, or order ID. Returns a paginated array of Payment objects."
---

You can retrieve a paginated list of all payments associated with your merchant account. Use the available query parameters to narrow results by payment status, creation date range, or a specific order ID. Results are returned in reverse chronological order — most recent first — and each page contains up to 100 payments. This endpoint is well-suited for building dashboards, running reconciliation reports, or exporting transaction history.

<Note>
  Date filters use the payment's `created_at` timestamp. If you need to filter by settlement date, download a full export from your PayMatrix dashboard instead.
</Note>

## Endpoint

```
GET https://api.paymatrix.com/v1/payments
```

## Query Parameters

<ParamField query="page" type="integer">
  The page number to retrieve. Defaults to `1`. Use together with `per_page` and the `pagination` object in the response to navigate through results.
</ParamField>

<ParamField query="per_page" type="integer">
  The number of payments to return per page. Defaults to `20`. Maximum allowed value is `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter results to payments with a specific status. Accepted values: `pending`, `succeeded`, `failed`, `cancelled`. Omit this parameter to return payments of all statuses.
</ParamField>

<ParamField query="from" type="string">
  An ISO 8601 datetime string defining the start of the date range filter — e.g. `2024-01-01T00:00:00Z`. Only payments created at or after this timestamp are included.
</ParamField>

<ParamField query="to" type="string">
  An ISO 8601 datetime string defining the end of the date range filter — e.g. `2024-01-31T23:59:59Z`. Only payments created at or before this timestamp are included.
</ParamField>

<ParamField query="order_id" type="string">
  Filter results to payments associated with a specific order ID — e.g. `ord_1A2B3C4D5E`.
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl "https://api.paymatrix.com/v1/payments?status=succeeded&from=2024-01-01T00:00:00Z&per_page=50" \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```

```python Python
import paymatrix

client = paymatrix.Client(api_key="pm_test_xxxxxxxxxxxx")

payments = client.payments.list(
    status="succeeded",
    from_date="2024-01-01T00:00:00Z",
    per_page=50,
)
```

```javascript Node.js
import PayMatrix from "paymatrix";

const client = new PayMatrix("pm_test_xxxxxxxxxxxx");

const payments = await client.payments.list({
  status: "succeeded",
  from: "2024-01-01T00:00:00Z",
  per_page: 50,
});
```

</CodeGroup>

## Response — 200 OK

A successful request returns a `200 OK` status with a `data` array of Payment objects and a `pagination` object describing the current page position.

```json Example Response
{
  "data": [
    {
      "id": "pay_9Z8Y7X6W5V",
      "order_id": "ord_1A2B3C4D5E",
      "status": "succeeded",
      "amount": 5000,
      "currency": "USD",
      "method": "card",
      "created_at": "2024-01-15T10:30:05Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 50,
    "total": 1,
    "total_pages": 1
  }
}
```

## Response Fields

<ResponseField name="data" type="array">
  An array of Payment objects matching your query. Each object contains a summary of the payment — retrieve an individual payment by ID for the full detail set, including card info and refunds.
</ResponseField>

<ResponseField name="pagination" type="object">
  Metadata about the current page of results.

  <Expandable title="pagination fields">
    <ResponseField name="page" type="integer">
      The current page number.
    </ResponseField>

    <ResponseField name="per_page" type="integer">
      The number of items returned per page, as requested.
    </ResponseField>

    <ResponseField name="total" type="integer">
      The total number of payments matching your query across all pages.
    </ResponseField>

    <ResponseField name="total_pages" type="integer">
      The total number of pages available at the current `per_page` size.
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  To iterate through all pages programmatically, increment `page` by `1` on each request until `page` equals `total_pages`. Alternatively, check whether the length of the `data` array is less than `per_page` to detect the final page.
</Tip>
