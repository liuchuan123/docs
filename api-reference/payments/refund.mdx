---
title: "POST /payments/{id}/refund — Refund a Payment"
sidebarTitle: "Refund Payment"
description: "Initiate a full or partial refund for a succeeded payment. Specify an amount for partial refunds. Returns the Refund object with pending status."
---

You can issue a refund against any payment that has a `succeeded` status. Omitting the `amount` field triggers a full refund for the original payment amount. To refund only a portion of the charge, pass an `amount` value in the smallest currency unit — for example, `2500` to refund $25.00 of a $50.00 payment. Multiple partial refunds can be issued against a single payment as long as the cumulative refunded amount does not exceed the original charge. Refunds are returned to the original payment method and typically reach the customer within 5–10 business days.

<Note>
  You can only refund payments with a `succeeded` status. Attempting to refund a `pending` or `failed` payment returns a `422 Unprocessable Entity` error.
</Note>

## Endpoint

```
POST https://api.paymatrix.com/v1/payments/{id}/refund
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the payment you want to refund — e.g. `pay_9Z8Y7X6W5V`.
</ParamField>

## Request Body

<ParamField body="amount" type="integer">
  The amount to refund in the smallest unit of the payment's currency — e.g. cents for USD. Omit this field entirely to issue a full refund for the original payment amount. Must be greater than `0` and must not exceed the remaining refundable balance on the payment.
</ParamField>

<ParamField body="reason" type="string">
  A machine-readable reason code for the refund. Accepted values:

  - `customer_request` — The customer asked to cancel or return their order.
  - `duplicate` — The payment was charged more than once in error.
  - `fraudulent` — The payment is suspected to be unauthorized.

  This field is optional but recommended — it helps PayMatrix optimize dispute handling on your behalf.
</ParamField>

<ParamField body="metadata" type="object">
  An optional set of key-value pairs to attach to the refund for your own record-keeping. Keys and values must be strings. Up to 20 keys are permitted, with values up to 500 characters each.
</ParamField>

## Example Requests

<CodeGroup>

```bash Full Refund
curl -X POST https://api.paymatrix.com/v1/payments/pay_9Z8Y7X6W5V/refund \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"reason": "customer_request"}'
```

```bash Partial Refund
curl -X POST https://api.paymatrix.com/v1/payments/pay_9Z8Y7X6W5V/refund \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"amount": 2500, "reason": "customer_request"}'
```

```python Python
import paymatrix

client = paymatrix.Client(api_key="pm_test_xxxxxxxxxxxx")

# Full refund
refund = client.payments.refund(
    "pay_9Z8Y7X6W5V",
    reason="customer_request",
)

# Partial refund
refund = client.payments.refund(
    "pay_9Z8Y7X6W5V",
    amount=2500,
    reason="customer_request",
)
```

```javascript Node.js
import PayMatrix from "paymatrix";

const client = new PayMatrix("pm_test_xxxxxxxxxxxx");

// Full refund
const refund = await client.payments.refund("pay_9Z8Y7X6W5V", {
  reason: "customer_request",
});

// Partial refund
const partialRefund = await client.payments.refund("pay_9Z8Y7X6W5V", {
  amount: 2500,
  reason: "customer_request",
});
```

</CodeGroup>

<Tip>
  Refunds are processed asynchronously. The returned `status` will initially be `pending`. Listen for the `refund.succeeded` or `refund.failed` webhook events to confirm the final outcome without polling.
</Tip>

## Response — 201 Created

A successful request returns a `201 Created` status and a Refund object. The refund begins in a `pending` state while PayMatrix processes it with the payment network.

```json Example Response
{
  "id": "ref_RRRRRRR",
  "payment_id": "pay_9Z8Y7X6W5V",
  "status": "pending",
  "amount": 2500,
  "currency": "USD",
  "reason": "customer_request",
  "metadata": {},
  "created_at": "2024-01-15T11:00:00Z"
}
```

## Response Fields

<ResponseField name="id" type="string">
  The unique identifier for this refund, prefixed with `ref_`.
</ResponseField>

<ResponseField name="payment_id" type="string">
  The ID of the payment this refund was issued against.
</ResponseField>

<ResponseField name="status" type="string">
  The current refund status. Newly created refunds always start as `pending`. Final values are `succeeded` or `failed`.
</ResponseField>

<ResponseField name="amount" type="integer">
  The refunded amount in the smallest unit of the currency — e.g. cents for USD. For a full refund, this matches the original payment `amount`.
</ResponseField>

<ResponseField name="currency" type="string">
  The three-letter ISO 4217 currency code — e.g. `USD`. Always matches the currency of the original payment.
</ResponseField>

<ResponseField name="reason" type="string">
  The reason code provided at the time of the refund request. Returns `null` if no reason was supplied.
</ResponseField>

<ResponseField name="metadata" type="object">
  The key-value metadata you attached to the refund request, or an empty object if none was provided.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the refund was created.
</ResponseField>
