Webhooks let you receive real-time HTTP notifications when events occur in your PayMatrix account, so you don't have to poll the API for status changes. When a subscribed event fires, PayMatrix sends an HTTP `POST` request to your registered URL containing a structured JSON payload describing what happened. You can register multiple webhook endpoints and scope each one to different event types depending on how your system is organized.

## Register a Webhook Endpoint

```
POST https://api.paymatrix.com/v1/webhooks
```

### Request Body

<ParamField body="url" type="string" required>
  The publicly accessible HTTPS URL where PayMatrix will deliver event payloads. HTTP URLs are not accepted — your endpoint must use TLS. The URL must respond with a `2xx` status code within 10 seconds to be considered a successful delivery.
</ParamField>

<ParamField body="events" type="array of strings" required>
  An array of event type strings you want to subscribe to (e.g., `["payment.succeeded", "refund.created"]`). To subscribe to **all** current and future event types, pass `["*"]`. See [Event Types](/api-reference/webhooks/events) for the full list.
</ParamField>

<ParamField body="description" type="string">
  A human-readable label to help you identify this webhook in the dashboard (e.g., `"Production webhook"` or `"Fulfillment service"`). Maximum 100 characters.
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X POST https://api.paymatrix.com/v1/webhooks \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourapp.com/webhooks/paymatrix",
    "events": ["payment.succeeded", "payment.failed", "refund.created"],
    "description": "Production webhook"
  }'
```
</CodeGroup>

### Response — 201 Created

```json
{
  "id": "whk_WWHH1122",
  "url": "https://yourapp.com/webhooks/paymatrix",
  "events": ["payment.succeeded", "payment.failed", "refund.created"],
  "description": "Production webhook",
  "secret": "whsec_SSSSSSSSSSSSSS",
  "created_at": "2024-01-15T10:00:00Z"
}
```

<Warning>
  The `secret` field is **returned only once**, at the moment the webhook is created. Copy and store it securely in your secrets manager immediately — it will not be shown again. This secret is used to verify the `X-PayMatrix-Signature` header on every incoming webhook request. If you lose it, you must delete and re-register the webhook to obtain a new secret.
</Warning>

## List Registered Webhooks

Use this endpoint to view all webhook endpoints currently registered on your account.

```
GET https://api.paymatrix.com/v1/webhooks
```

### Request Example

<CodeGroup>
```bash cURL
curl https://api.paymatrix.com/v1/webhooks \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```
</CodeGroup>

### Response — 200 OK

```json
{
  "data": [
    {
      "id": "whk_WWHH1122",
      "url": "https://yourapp.com/webhooks/paymatrix",
      "events": ["payment.succeeded", "payment.failed", "refund.created"],
      "description": "Production webhook",
      "created_at": "2024-01-15T10:00:00Z"
    }
  ]
}
```

<Note>
  The `secret` is intentionally omitted from all `GET` responses. Only the registration response (`POST /webhooks`) exposes the secret.
</Note>

## Remove a Webhook Endpoint

Delete a registered webhook to permanently stop PayMatrix from delivering events to that URL.

```
DELETE https://api.paymatrix.com/v1/webhooks/{id}
```

### Request Example

<CodeGroup>
```bash cURL
curl -X DELETE https://api.paymatrix.com/v1/webhooks/whk_WWHH1122 \
  -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
```
</CodeGroup>

### Response — 204 No Content

A successful deletion returns a `204 No Content` response with an empty body. Event delivery to the deleted endpoint will stop immediately.
