根据**平台订单号（`order_id`）**查询支付订单的当前状态和详细信息。

## 接口信息

| 项目         | 说明                                                |
| ------------ | --------------------------------------------------- |
| 请求方式     | `GET`                                               |
| 路径         | `/api/merchant/order/payment/query/{orderId}`       |
| Content-Type | —（GET 请求无 Body）                                |
| 鉴权方式     | RSA 签名（签名原文中 Body 部分为空行）              |

## 路径参数

| 参数      | 类型   | 必填 | 说明                                             |
| --------- | ------ | ---- | ------------------------------------------------ |
| `orderId` | String | 是   | 平台订单号（创建支付接口返回的 `order_id`）      |

<Note>
  查询使用的是**平台订单号 `order_id`**，不是商户订单号 `merchant_order_no`。请保存创建支付时返回的 `order_id`。
</Note>

## 响应参数

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "platform_order_id": "26020100012345678",
    "merchant_order_no": "ORD-20260201-0001",
    "payment_currency_code": "USD",
    "payment_amount": 100.00,
    "payment_status": 2,
    "payment_success_time": "2026-02-01 12:30:00",
    "settlement_currency_code": "USD",
    "settlement_amount": 98.00
  }
}
```

### 字段说明

| 字段                       | 类型       | 说明                                         |
| -------------------------- | ---------- | -------------------------------------------- |
| `platform_order_id`        | String     | 平台订单号                                   |
| `merchant_order_no`        | String     | 商户订单号                                   |
| `payment_currency_code`    | String     | 支付币种                                     |
| `payment_amount`           | BigDecimal | 支付金额                                     |
| `payment_status`           | Integer    | 订单维度支付状态（见下方枚举）               |
| `payment_success_time`     | String     | 支付成功时间，格式 `yyyy-MM-dd HH:mm:ss`     |
| `settlement_currency_code` | String     | 结算币种（可能与支付币种不同）               |
| `settlement_amount`        | BigDecimal | 结算金额（扣除平台手续费后）                 |

### 支付状态枚举

`payment_status` 为**订单维度**状态（参照 `OrderStatusEnum`）：

| 状态码 | 状态           | 说明                 |
| ------ | -------------- | -------------------- |
| 0      | 待支付         | 订单已创建，等待支付 |
| 1      | 支付中         | 用户正在进行支付     |
| 2      | 支付成功       | 支付已完成           |
| 3      | 支付失败       | 支付处理失败         |
| 4      | 部分退款       | 已退款部分金额       |
| 5      | 已退款         | 订单金额全部退还     |
| 6      | 已关闭         | 主动取消或关闭       |
| 7      | 争议中         | 发生 Chargeback      |
| 8      | 已过期         | 超时未支付过期       |

<Tip>
  对账请以订单维度的 `payment_status` 为准。单笔交易尝试另有独立状态机（`TransactionStatusEnum`），口径不同，不要混用。详见 [订单生命周期](/concepts/order-lifecycle)。
</Tip>

## 使用场景

- **后台对账**：定时拉取订单状态与本地系统核对
- **状态排查**：Webhook 未收到或延迟时，手动查询确认
- **补单**：下单超时未拿到响应时，确认订单是否已创建

<Tip>
  建议以 **Webhook 通知** 作为状态变更的主要驱动，支付查询接口仅作为辅助验证与异常排查使用，避免高频轮询。
</Tip>
