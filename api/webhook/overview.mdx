Webhook 是 PayMatrix 向商户服务端推送实时事件通知的机制。当支付、退款、争议等关键事件发生时，平台会主动向商户配置的回调地址发送 HTTP POST 请求。

## 为什么需要 Webhook

<CardGroup cols={3}>
  <Card title="实时性" icon="bolt">
    支付状态变更后秒级通知，无需轮询
  </Card>
  <Card title="可靠性" icon="shield-check">
    平台保证至少投递一次，支持失败重试
  </Card>
  <Card title="权威性" icon="badge-check">
    Webhook 是订单状态变更的权威来源
  </Card>
</CardGroup>

## 回调请求格式

### HTTP 方法

`POST`，Content-Type 为 `application/json`

### 请求头

| Header               | 说明                         |
| -------------------- | ---------------------------- |
| `Content-Type`       | `application/json`           |
| `X-Merchant-Id`      | 商户 ID                      |
| `X-Pmp-Event-Id`     | 事件唯一 ID（如 `evt_xxx`）  |
| `X-Pmp-Event-Type`   | 事件类型（如 `order.payment.succeeded`） |
| `X-Pmp-Timestamp`    | Unix 秒级时间戳              |
| `X-Pmp-Signature`    | HMAC-SHA256 签名（用于验签） |

### 请求体结构

```json
{
  "event_id": "evt_order.payment.succeeded_10001_26020100012345678",
  "event_type": "order.payment.succeeded",
  "timestamp": 1749081600,
  "data": {
    "order_id": "26020100012345678",
    "transaction_id": "26020100055500001",
    "merchant_order_no": "ORD-20260201-0001",
    "amount": 100.00,
    "currency": "USD",
    "settlement_amount": 98.00,
    "settlement_currency": "USD",
    "platform_fee": 2.00,
    "payment_method": "card",
    "success_time": "2026-02-01 12:30:00"
  }
}
```

| 字段         | 类型   | 说明                           |
| ------------ | ------ | ------------------------------ |
| `event_id`   | String | 事件唯一 ID，用于幂等去重      |
| `event_type` | String | 事件类型                       |
| `timestamp`  | Long   | 事件发生时间（Unix **秒**）    |
| `data`       | Object | 事件负载，字段随事件类型变化，详见 [事件类型](/api/webhook/events) |

## 配置 Webhook

登录商户门户，进入「开发设置 → Webhook 管理」：

1. 点击「添加 Webhook」
2. 填写你的回调 URL（**必须 HTTPS**）
3. 设置签名密钥（用于验签，可自动生成）
4. 勾选需要订阅的事件类型
5. 保存

<Tip>
  可为不同用途（如测试与生产）分别配置不同的 Webhook URL，便于区分事件来源。
</Tip>

## 测试 Webhook

配置完 Webhook 后，可以使用「测试发送」功能验证：

1. 在 Webhook 列表中选择要测试的 Webhook
2. 点击「测试发送」
3. 选择事件类型
4. 可选填写自定义 Payload
5. 点击发送

平台会向你的回调地址发送一条测试通知，你可以查看投递结果。

## 投递重试机制

- 首次投递失败后，平台会按策略自动重试
- **指数退避**（默认策略）：重试间隔依次为 1min、3min、9min、27min、81min、243min（每次 ×3）
- **固定间隔**（可选策略）：每次固定 5min
- 最大重试次数默认 **6** 次，超过后标记为「已达最大重试」并将该 Webhook 标记为异常
- 单次投递超时默认 **10 秒**
- 每个 Webhook 的最大重试次数、重试策略、超时时间均可在门户配置
- 可在商户门户查看投递记录、审计日志，并对失败投递手动重试

## 商户服务端要求

- **响应时间**：请在 10 秒内返回 HTTP 2xx 状态码，否则视为投递失败
- **幂等处理**：同一 `event_id` 可能被重复投递，务必做幂等处理
- **签名验证**：收到回调后先验签，确认请求来自 PayMatrix
