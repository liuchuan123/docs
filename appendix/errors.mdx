# 错误码参考

本文列出了 OpenAPI 可能返回的常见错误码及处理建议。

## 通用响应格式

所有接口在出错时返回统一格式：

```json
{
  "code": 40001,
  "msg": "参数校验失败：amount 必须大于 0",
  "data": null
}
```

## 错误码分类

### 签名与鉴权错误

| 错误码 | 错误信息               | 说明                         | 处理建议                     |
| ------ | ---------------------- | ---------------------------- | ---------------------------- |
| 40101  | MISSING_HEADER         | 缺少必填的签名请求头         | 检查 4 个签名头是否全部携带   |
| 40102  | TIMESTAMP_EXPIRED      | 时间戳超出有效窗口（±2分钟） | 校准服务器时间               |
| 40103  | SIGNATURE_INVALID      | RSA 签名验证失败             | 检查签名算法和私钥是否正确   |
| 40104  | NONCE_REPLAY           | Nonce 重复使用               | 确保每次请求生成新的随机串   |
| 40105  | MERCHANT_NOT_FOUND     | 商户 ID 不存在               | 检查 X-Merchant-Id 是否正确  |
| 40106  | IP_NOT_IN_WHITELIST    | 客户端 IP 不在白名单内       | 将当前服务器 IP 加入白名单   |

### 参数校验错误

| 错误码 | 错误信息                 | 说明                   |
| ------ | ------------------------ | ---------------------- |
| 40001  | PARAM_INVALID            | 请求参数校验失败       |
| 40002  | AMOUNT_INVALID           | 金额格式错误或超出范围 |
| 40003  | CURRENCY_NOT_SUPPORTED   | 不支持的币种           |
| 40004  | PRODUCTS_EMPTY           | 商品列表为空           |
| 40005  | REDIRECT_URL_INVALID     | 跳转地址格式错误       |

### 业务错误

| 错误码 | 错误信息                   | 说明                           |
| ------ | -------------------------- | ------------------------------ |
| 40201  | ORDER_NOT_FOUND            | 订单不存在                     |
| 40202  | ORDER_DUPLICATE            | 商户交易号重复                 |
| 40203  | ORDER_EXPIRED              | 订单已过期，无法支付           |
| 40204  | ORDER_ALREADY_PAID         | 订单已支付，不可重复支付       |
| 40205  | ORDER_CLOSED               | 订单已关闭                     |
| 40206  | REFUND_EXCEED_AMOUNT       | 退款金额超出可退金额           |
| 40207  | REFUND_NOT_ALLOWED         | 当前订单状态不允许退款         |
| 40208  | REFUND_DUPLICATE           | 退款交易号重复                 |
| 40209  | REFUND_IN_DISPUTE          | 订单存在争议中，暂不可退款     |

### 系统错误

| 错误码 | 错误信息             | 说明               | 处理建议           |
| ------ | -------------------- | ------------------ | ------------------ |
| 50001  | SYSTEM_ERROR         | 系统内部错误       | 稍后重试           |
| 50002  | CHANNEL_ERROR        | 支付渠道异常       | 稍后重试或切换渠道 |
| 50003  | SERVICE_UNAVAILABLE  | 服务暂不可用       | 稍后重试           |
| 50004  | TIMEOUT              | 请求处理超时       | 稍后重试           |

## 重试策略

对于可重试的错误（如系统错误、渠道异常），建议采用以下策略：

| 类型           | 建议                               |
| -------------- | ---------------------------------- |
| 幂等接口       | 相同参数重试（平台保证幂等）       |
| 非幂等接口     | 先查询状态确认，再决定是否重试     |
| 重试间隔       | 1s → 3s → 5s（指数退避）          |
| 最大重试次数   | 3 次                               |

## 错误处理最佳实践

1. **记录日志**：记录每次 API 调用的请求和响应，方便排查问题
2. **根据错误码分类处理**：
   - 4xx 错误：检查请求参数，修正后重试
   - 5xx 错误：等待后重试，若持续出现则联系技术支持
3. **签名错误**：首先检查时间同步和签名算法实现
4. **提供降级方案**：支付异常时不应阻塞用户体验
