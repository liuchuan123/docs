---
title: "收银台"
description: "PayMatrix 托管收银台的接入方式与回跳处理"
---

收银台由 PayMatrix 统一托管。商户创建支付订单后，只需把用户浏览器跳转到接口返回的 `pay_url`，其余支付流程（支付方式选择、收银界面、卡信息采集、渠道要求的认证、结果展示）由平台完成。

<Tip>
  **无需采集卡信息**：你不需要在自己的页面处理原始卡号，只做「跳转」和「接收结果」。你的具体合规义务仍以签约协议和业务场景为准。
</Tip>

## 集成流程

<Steps>
  <Step title="用户下单">
    用户在商户网站选择商品并提交订单。
  </Step>
  <Step title="创建支付订单">
    商户服务端调用 <a href="/api/payment/create">POST /api/merchant/payment/create</a>，获取 <code>order_id</code> 与 <code>pay_url</code>。
  </Step>
  <Step title="跳转收银台">
    将用户浏览器重定向到 <code>pay_url</code>。
  </Step>
  <Step title="用户完成支付">
    用户在托管收银台选择支付方式并完成付款，平台自动处理收银界面与认证。
  </Step>
  <Step title="回跳商户">
    支付结束后跳转 <code>redirect_url</code>（携带结果参数），取消则跳转 <code>cancel_url</code>。
  </Step>
  <Step title="Webhook 确认">
    平台推送 <code>order.payment.succeeded</code> 等事件，商户以此更新订单状态。
  </Step>
</Steps>

## 第一步：创建支付订单

商户服务端调用 OpenAPI 创建支付订单，并在参数中带上回跳地址：

- `redirect_url`：支付成功后用户回跳的地址
- `cancel_url`：用户取消支付时回跳的地址

详见 [创建支付](/api/payment/create)。响应中包含 `pay_url`：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "order_id": "26020100012345678",
    "pay_url": "https://cashier.paymatrixpay.com/checkout?orderId=26020100012345678"
  }
}
```

## 第二步：跳转到收银台

将用户浏览器重定向到 `pay_url`：

```javascript
// 前端跳转
window.location.href = payUrl;
```

或服务端返回 302：

```
HTTP 302 Location: https://cashier.paymatrixpay.com/checkout?orderId=26020100012345678
```

## 第三步：用户在收银台完成支付

用户进入收银台后会看到订单摘要、可用支付方式与对应的支付界面。平台自动处理：

- 支付方式呈现与切换
- 卡信息采集（PCI 合规由平台负责）
- 渠道要求的认证流程（如适用）
- 支付结果与超时展示

## 第四步：处理支付结果

### 用户侧（同步回跳）

支付结束后，用户被回跳到 `redirect_url`，URL 上会附带以下结果参数：

| 参数               | 说明                                             |
| ------------------ | ------------------------------------------------ |
| `p_transaction_id` | 平台交易号                                       |
| `p_status`         | 交易状态（`2` 成功、`6` 失败、`1` 处理中 等）    |

示例：

```
https://merchant.com/payment/success?p_transaction_id=26020100055500001&p_status=2
```

<Warning>
  回跳参数仅用于前端展示，**不可作为发货或入账依据**。用户可能中途关闭页面导致不回跳，回跳也可能被伪造。请始终以 [Webhook 通知](/api/webhook/overview) 作为订单状态的权威来源。
</Warning>

### 服务端（Webhook，权威）

平台异步推送 Webhook 通知，商户据此更新订单：

```json
{
  "event_type": "order.payment.succeeded",
  "data": {
    "order_id": "26020100012345678",
    "merchant_order_no": "ORD-20260201-0001",
    "amount": 100.00,
    "currency": "USD"
  }
}
```

收到通知后：验签 → 用 `merchant_order_no` 匹配本地订单 → 更新状态（如待发货 → 已支付）→ 返回 HTTP 200。

## 收银台定制

| 定制项    | 说明                       | 配置位置            |
| --------- | -------------------------- | ------------------- |
| 商户 Logo | 收银台顶部展示的商户 Logo  | 商户门户 → 企业信息 |
| 语言      | 多语言，自动适配浏览器语言 | 自动                |

## 安全说明

- **地址安全**：Webhook 地址必须使用 HTTPS；`redirect_url` 与 `cancel_url` 需为合法 URL，生产环境强烈建议使用 HTTPS
- **订单有效期**：`pay_url` 有时效，过期后需重新创建订单
- **以 Webhook 为准**：不要仅凭前端回跳判断支付结果
