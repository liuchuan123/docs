PayMatrix 为每个商户维护独立的账户与流水。每一笔支付、退款都会生成可追溯的账本记录，并沉淀到商户账户的余额中，便于对账与结算。

## 资金链路

```
支付成功 → 商户账本（逐笔入账） → 商户账户余额 → 结算单（释放到可提现） → 提现到银行
```

平台完成支付处理与对账后，将资金按币种记入商户账户；底层如何清算对商户透明。

## 商户账户余额

每个商户按币种维护四类余额（详见 [结算体系](/concepts/settlement)）：

| 余额     | 说明                             |
| -------- | -------------------------------- |
| 待结算   | 已支付入账、尚未结算释放的金额   |
| 可提现   | 已结算释放、可发起提现的金额     |
| 冻结     | 因争议等被冻结的金额             |
| 保证金   | 预留的风险保证金（设有上限）     |

## 商户账本

商户账本记录商户维度的每一笔资金变动，常见类型包括：

- **支付入账**：支付成功后进入待结算（已扣平台手续费与保证金）
- **退款**：以负数计入，抵减待结算/可提现
- **结算释放**：待结算 → 可提现的内部过户
- **保证金计提/释放**

每条账本记录都关联到对应的交易或结算，可逐笔溯源。

## 对账

商户可从以下维度自助对账，确保本地系统与平台一致：

- **交易对账**：商户交易记录与平台记录逐笔核对
- **结算对账**：商户待结算/可提现金额与结算单核对

<Tip>
  建议每日进行交易对账。可结合 [支付查询](/api/payment/query)、[退款查询](/api/payment/query-refund) 接口，以及门户的交易/结算导出进行核对。
</Tip>
