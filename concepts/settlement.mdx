支付成功后，资金并非立即可提现，而是先进入**待结算**，经平台对账、结算入账后转为**可提现**余额，再由你发起提现到银行账户。

## 资金流转总览

```mermaid
flowchart LR
    A[支付成功] --> B[待结算余额]
    B -->|对账 + 结算入账| C[可提现余额]
    C -->|提现| D[银行账户]
```

<Note>
  平台的「结算」指账户内部的资金释放（待结算户 → 可提现户），并非直接向银行打款。资金到达可提现余额后，需由你在门户主动发起**提现**才会出款到银行账户。
</Note>

## 账户余额类型

每个商户按币种维护四类余额：

| 余额类型   | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 待结算余额 | 支付成功入账、但尚未完成结算释放的金额                       |
| 可提现余额 | 已结算释放、可随时发起提现的金额                             |
| 冻结余额   | 因争议等原因被冻结的金额，处理完成后解冻                     |
| 保证金     | 预留的风险保证金，通常按交易额的一定比例计提，设有上限       |

## 手续费与入账金额

<Info>
  平台手续费与保证金在**交易入账时**即已扣除，而非等到结算日。
</Info>

一笔支付成功后，进入待结算的金额为：

```
待结算金额 = 支付金额 − 平台手续费 − 保证金计提
```

- **平台手续费（platform_fee）**：PayMatrix 服务费，商户可见，计入结算明细。
- **保证金计提**：按比例进入保证金账户，风险期过后可释放。
- 平台承担的上游处理成本**不向商户收取**，商户仅需关注平台手续费。

支付查询接口返回的 `settlement_amount` 即为扣除平台手续费后的结算金额。

## 结算周期

平台采用**基于实际到账 + 逐笔释放日**的结算模型，而非固定的统一周期：

- 结算批次按平台**实际收到清算资金的日期**归集。
- 每笔入账另有**预计释放日（T+N）**，到期后方可从待结算释放到可提现。

<Tip>
  具体的费率、保证金比例与释放账期以你的签约协议为准。如需调整，请联系客户经理。
</Tip>

## 结算单

平台按「商户 + 币种 + 结算日」生成结算单（Settlement），经运营审核后完成内部入账。结算单主要包含：

| 字段           | 说明                                             |
| -------------- | ------------------------------------------------ |
| 结算单号       | 唯一编号，形如 `MST-{日期}-{商户ID}-{币种}`      |
| 结算日         | 本次结算的归集日期                               |
| 原始金额合计   | 本期交易本金合计                                 |
| 结算净额       | 实际释放到可提现的净额（已扣手续费、含退款抵减） |
| 平台手续费合计 | 本期平台手续费总额                               |
| 保证金合计     | 本期计提的保证金                                 |
| 交易笔数       | 本期结算覆盖的交易笔数                           |
| 状态           | 见下方状态枚举                                   |

退款以**负数明细**计入本期结算，因此结算净额已抵减本期发生的退款。

### 结算单状态

| 状态码 | 状态       | 说明                             |
| ------ | ---------- | -------------------------------- |
| 1      | 待运营审核 | 结算单已生成，等待平台审核       |
| 2      | 已结算     | 审核通过，已完成内部入账（释放到可提现）|
| 3      | 已驳回     | 审核未通过                       |
| 4      | 失败       | 结算入账失败                     |

<Note>
  资金释放为**运营审核后触发**，不会在生成结算单时自动放款。
</Note>

## 提现

可提现余额可在门户「结算管理」中发起提现：

1. 选择币种与提现金额
2. 填写收款银行账户信息（户名、账号、银行名称）
3. 提交申请

<Tip>
  提现时填写收款银行账户即可，**无需预先绑定银行卡**。
</Tip>
