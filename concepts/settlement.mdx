## 结算流程

支付成功后，资金并非立即到达你的银行账户，而是经过以下结算流程：

```
消费者付款 → 支付渠道 → PayMatrix → 待结算余额 → 结算日 → 商户账户余额 → 提现 → 银行账户
```

### 各环节说明

1. **消费者付款**：消费者通过银行卡/钱包支付
2. **支付渠道清算**：Visa/Stripe/Alipay 等渠道方处理资金
3. **平台到账**：资金到达 PayMatrix 平台
4. **待结算**：资金进入商户的待结算余额
5. **结算执行**：到达结算日，资金从待结算转入商户账户余额
6. **提现**：商户将账户余额提现到指定银行卡

<Note>
  结算资金进入平台分配的**商户账户（merchant_account）**，一个商户可拥有多个账户。提现时临时指定收款银行卡号，无需预先绑定。
</Note>

## 结算周期

| 周期       | 说明                             | 适用场景     |
| ---------- | -------------------------------- | ------------ |
| T+1        | 交易次日结算                     | 标准商户     |
| T+7        | 交易后第 7 天结算                | 新商户/高风险 |
| 周结       | 每周固定日结算上一周的交易       | 大额交易     |
| 月结       | 每月固定日结算上一个月的交易     | 特殊行业     |

<Tip>
  具体结算周期以签约协议为准。新入驻商户通常为 T+7，稳定运营后可调整为 T+1。
</Tip>

## 结算单

每次结算会生成一个结算批次（Settlement Batch），包含：

- **结算批次号**：唯一标识
- **结算周期**：本次结算覆盖的交易时间范围
- **交易笔数/金额**：本期成功交易汇总
- **手续费**：平台手续费、渠道手续费
- **退款扣减**：本期发生的退款金额
- **净结算金额**：实际打款金额 = 交易金额 - 手续费 - 退款

## 手续费

交易手续费在结算时扣除，包含：

| 费用类型   | 说明               |
| ---------- | ------------------ |
| 渠道手续费 | 支付渠道收取的费用 |
| 平台手续费 | PayMatrix 服务费 |

每笔交易的手续费可以在交易详情中查看。

## 账户余额类型

| 账户       | 说明                                         |
| ---------- | -------------------------------------------- |
| 待结算余额 | 已支付成功但尚未到结算日的金额               |
| 可用余额   | 已完成结算、可提现的金额                     |
| 冻结余额   | 因争议等原因被冻结的金额，争议解决后解冻     |
| 保证金     | 预留风险保证金，通常为交易额的固定比例       |

## 结算报表

商户可导出结算报表用于财务对账，包含：
- 交易明细（每笔交易的金额、手续费）
- 退款扣减明细
- 结算汇总

支持按日/周/月导出为 CSV 或 Excel 格式。
