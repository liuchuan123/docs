本章节帮助你快速了解从入驻 PayMatrix 到完成第一笔支付的完整流程。

<Note>
  **10 分钟接入路径**：① 用 [签名工具与示例](/integration/sdk) 复制一份签名代码 → ② [创建支付](/api/payment/create) 拿到 `pay_url` → ③ 跳转 [收银台](/checkout/hosted) 完成付款 → ④ 用 [Webhook](/api/webhook/overview) 接收结果。直接看下方「第三步：完成第一笔支付」。
</Note>

  </Card>

## 接入流程总览

<Steps>
  <Step title="商户入驻">
    在商户门户注册并提交资料，等待平台审核通过。
  </Step>
  <Step title="配置密钥">
    生成 RSA 密钥对，将公钥上传至门户「开发设置」，并记录商户 ID。
  </Step>
  <Step title="配置 Webhook">
    设置接收事件通知的回调地址和签名密钥。
  </Step>
  <Step title="联调测试">
    调用 API 验证签名与业务流程（测试商户由平台开通）。
  </Step>
  <Step title="集成收银台">
    使用托管收银台或自建方案完成前端支付流程。
  </Step>
  <Step title="上线">
    完成联调后正式对外提供服务。
  </Step>
</Steps>

## 环境与地址

| 用途     | 地址                                     |
| -------- | ---------------------------------------- |
| OpenAPI  | `https://openapi.paymatrixpay.com/api`   |
| 收银台   | `https://cashier.paymatrixpay.com`       |
| 商户门户 | `https://merchant.paymatrixpay.com`      |

OpenAPI 接口前缀：
- 支付接口：`/api/merchant/payment`
- 订单查询：`/api/merchant/order`

## 第一步：注册商户账号

前往 [商户门户](https://merchant.paymatrixpay.com) 注册账号并提交入驻资料，等待审核通过。详见 [入驻指南](/onboarding/overview)。

## 第二步：配置 API 密钥

审核通过后，登录商户门户进入「开发设置」：

1. **生成 RSA 密钥对**（2048 位，SHA256WithRSA）
2. **上传公钥**至平台
3. **记录商户 ID**（`merchantId`），后续所有 API 调用需在 `X-Merchant-Id` 请求头携带

<Tip>
  妥善保管私钥，不要泄露或提交到代码仓库。建议使用密钥管理服务（KMS）存储。
</Tip>

## 第三步：完成第一笔支付

### 1. 创建支付订单

```bash
curl -X POST "https://openapi.paymatrixpay.com/api/merchant/payment/create" \
  -H "Content-Type: application/json" \
  -H "X-Merchant-Id: YOUR_MERCHANT_ID" \
  -H "X-Timestamp: 1737004800000" \
  -H "X-Nonce: a3f7b2c1d4e5" \
  -H "X-Signature: <RSA_SIGNATURE>" \
  -d '{
    "merchant_order_no": "ORD-20260201-0001",
    "amount": 100.00,
    "currency_code": "USD",
    "redirect_url": "https://your-site.com/payment/return",
    "cancel_url": "https://your-site.com/payment/cancel",
    "products": [
      {
        "product_id": "PROD001",
        "name": "Example Product",
        "price": 100.00,
        "quantity": 1,
        "url": "https://your-site.com/product/PROD001"
      }
    ],
    "shipping": {
      "name": "Jane Smith",
      "address": {
        "line1": "1 Market Street",
        "city": "San Francisco",
        "state": "CA",
        "country": "US",
        "postal_code": "94105"
      }
    },
    "customer": {
      "full_name": "John Doe"
    }
  }'
```

### 2. 引导用户跳转收银台

成功响应中包含 `order_id` 与 `pay_url`，将用户重定向到 `pay_url` 完成支付。

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "order_id": "26020100012345678",
    "pay_url": "https://cashier.paymatrixpay.com/checkout?orderId=26020100012345678"
  }
}
```

### 3. 接收 Webhook 通知

支付完成后，平台会向你的 Webhook 地址推送 `order.payment.succeeded` 事件。

<Warning>
  务必以 Webhook 通知作为订单状态变更的权威来源，前端重定向仅用于提升用户体验，不可作为发货依据。
</Warning>

### 4. 查询支付状态（可选）

用返回的平台订单号 `order_id` 查询：

```bash
curl -X GET "https://openapi.paymatrixpay.com/api/merchant/order/payment/query/26020100012345678" \
  -H "X-Merchant-Id: YOUR_MERCHANT_ID" \
  -H "X-Timestamp: 1737004800000" \
  -H "X-Nonce: a3f7b2c1d4e5" \
  -H "X-Signature: <RSA_SIGNATURE>"
```
