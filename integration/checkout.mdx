PayMatrix.js is a JavaScript library that renders a hosted payment form directly on your webpage, allowing you to collect card details from customers without ever touching sensitive card data on your own server. Because the form fields are securely hosted by PayMatrix, your integration is PCI-compliant by default — reducing your compliance scope significantly.

## Installation

You can load PayMatrix.js either via a CDN script tag or by installing the npm package, depending on your project setup.

<CodeGroup>
```html CDN
<script src="https://js.paymatrix.com/v1/paymatrix.js"></script>
```

```bash npm
npm install @paymatrix/js
```
</CodeGroup>

## Integration Steps

<Steps>
  ### Load PayMatrix.js

  Add the script tag to the `<head>` of your HTML page, or import the package at the top of your JavaScript/TypeScript module.

  <CodeGroup>
  ```html CDN
  <script src="https://js.paymatrix.com/v1/paymatrix.js"></script>
  ```

  ```javascript npm
  import PayMatrix from '@paymatrix/js';
  ```
  </CodeGroup>

  ### Create an Order Server-Side

  Before rendering the payment form, create an order on your server by calling `POST /v1/orders`. This returns an `orderId` that you pass to the client-side form. Never create orders directly from the browser, as this requires your secret API key.

  ```bash
  curl -X POST https://api.paymatrix.com/v1/orders \
    -H "Authorization: Bearer sk_test_xxxxxxxxxxxx" \
    -H "Content-Type: application/json" \
    -d '{
      "amount": 4999,
      "currency": "usd",
      "description": "Order #1234"
    }'
  ```

  The response includes an `id` field (e.g. `ord_1A2B3C4D5E`) that you forward to your frontend.

  ### Initialize the Checkout Form

  Add a container element to your HTML where the payment form will be mounted, then initialize PayMatrix.js with your publishable key and the order ID from the previous step.

  ```html
  <div id="payment-form"></div>
  ```

  ```javascript
  const pm = PayMatrix('pm_test_xxxxxxxxxxxx');

  const form = pm.createPaymentForm({
    orderId: 'ord_1A2B3C4D5E',
    container: '#payment-form',
    onSuccess: (payment) => {
      console.log('Payment succeeded:', payment.id);
      window.location.href = '/thank-you';
    },
    onError: (error) => {
      console.error('Payment failed:', error.message);
    }
  });

  form.mount();
  ```

  <Warning>
    Never log or store full card numbers. PayMatrix.js tokenizes card data in the browser before it ever reaches your server — your backend only receives a secure payment token.
  </Warning>

  ### Handle the Payment Result

  Use the `onSuccess` and `onError` callbacks to respond to the payment outcome. On success, redirect the customer to a confirmation page or update your order state. On error, display a meaningful message so the customer can retry.

  <Tip>
    Always confirm the final payment status server-side by listening for the `payment.succeeded` webhook event — do not rely solely on the client-side `onSuccess` callback for order fulfillment.
  </Tip>
</Steps>

## How It Works

<CardGroup cols={3}>
  <Card title="Hosted Fields" icon="shield-check">
    Card input fields are rendered inside a PayMatrix-hosted iframe, so sensitive data never touches your DOM or your server.
  </Card>
  <Card title="Tokenization" icon="lock">
    PayMatrix.js converts card details into a secure, single-use token before any network request is made to your backend.
  </Card>
  <Card title="PCI Compliance" icon="badge-check">
    Because card data is handled entirely by PayMatrix, your integration qualifies for the simplest PCI DSS compliance tier (SAQ A).
  </Card>
</CardGroup>
