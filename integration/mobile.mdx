The PayMatrix mobile SDKs give you native payment UI components for both iOS and Android, so you can present a polished, familiar payment experience without building card forms from scratch. Each SDK handles tokenization, 3D Secure authentication, and digital wallet support (Apple Pay and Google Pay) automatically, keeping your app's code free of sensitive card data and reducing your PCI compliance burden.

## Prerequisites

Before integrating the mobile SDK, make sure you have:

<CardGroup cols={2}>
  <Card title="A PayMatrix Account" icon="user">
    Sign up at [dashboard.paymatrix.com](https://dashboard.paymatrix.com) and retrieve your publishable API key from **Settings → API Keys**.
  </Card>
  <Card title="A Server-Side Order Endpoint" icon="server">
    Your backend must create a PayMatrix order via `POST /v1/orders` and return the `orderId` to your mobile app before presenting the payment sheet.
  </Card>
</CardGroup>

## SDK Installation & Usage

<Tabs>
  <Tab title="iOS">
    ### Install via Swift Package Manager

    Add the PayMatrix iOS SDK as a package dependency in your `Package.swift` file or through Xcode's **File → Add Packages** menu.

    ```swift
    // Package.swift
    dependencies: [
        .package(url: "https://github.com/paymatrix/paymatrix-ios", from: "1.0.0")
    ]
    ```

    ### Initialize and Present the Payment Sheet

    Import the SDK, create a `PayMatrixClient` with your publishable key, configure the payment sheet, and present it from any `UIViewController`.

    ```swift
    import PayMatrix

    let pm = PayMatrixClient(publishableKey: "pm_test_xxxxxxxxxxxx")

    let config = PaymentSheetConfig(
        orderId: "ord_1A2B3C4D5E",
        merchantName: "Acme Corp"
    )

    PaymentSheet.present(configuration: config, from: viewController) { result in
        switch result {
        case .completed(let payment):
            print("Payment succeeded: \(payment.id)")
        case .failed(let error):
            print("Payment failed: \(error.localizedDescription)")
        case .canceled:
            print("User canceled")
        }
    }
    ```

    ### Supported iOS Features

    - Apple Pay via PassKit
    - 3D Secure 2 challenge flows
    - Card scanning via the device camera
    - Dark mode and Dynamic Type support

    <Tip>
      Use the `.completed` result exclusively to trigger order fulfillment on your server. Confirm the payment status independently by verifying the `payment.succeeded` webhook event.
    </Tip>
  </Tab>

  <Tab title="Android">
    ### Install via Gradle

    Add the PayMatrix Android SDK to your app-level `build.gradle.kts` dependencies block.

    ```kotlin
    // build.gradle.kts
    dependencies {
        implementation("com.paymatrix:paymatrix-android:1.0.0")
    }
    ```

    Sync your project after adding the dependency so Android Studio can resolve the SDK classes.

    ### Launch the Payment Sheet

    Create a `PayMatrixClient`, build a `PaymentSheetConfig` with the order ID from your server, and launch the payment sheet from any `Activity` or `Fragment`.

    ```kotlin
    val client = PayMatrixClient(publishableKey = "pm_test_xxxxxxxxxxxx")

    val config = PaymentSheetConfig(
        orderId = "ord_1A2B3C4D5E",
        merchantName = "Acme Corp"
    )

    PaymentSheet.present(activity, config) { result ->
        when (result) {
            is PaymentResult.Completed -> Log.d("PM", "Payment: ${result.payment.id}")
            is PaymentResult.Failed -> Log.e("PM", result.error.message)
            is PaymentResult.Canceled -> Log.d("PM", "User canceled")
        }
    }
    ```

    ### Supported Android Features

    - Google Pay via the Google Pay API
    - 3D Secure 2 challenge flows
    - Material You theming
    - Jetpack Compose support via `PaymentSheet.presentComposable()`

    <Tip>
      Register the `PaymentSheet` result launcher in `onCreate` — before the activity is resumed — to avoid missed results on configuration changes.
    </Tip>
  </Tab>
</Tabs>

<Note>
  The mobile SDKs handle PCI compliance automatically — card data never passes through your app's code. All sensitive input is captured inside a PayMatrix-managed view and tokenized before your result callback fires.
</Note>

## Error Handling

Both SDKs surface errors through a typed result enum, so you can handle each case explicitly rather than catching generic exceptions.

| Result State | Meaning | Recommended Action |
|---|---|---|
| `completed` / `Completed` | Payment authorized successfully | Confirm server-side, then fulfill the order |
| `failed` / `Failed` | Payment was declined or errored | Display `error.localizedDescription` to the user |
| `canceled` / `Canceled` | User dismissed the sheet | Allow the user to retry at their discretion |
