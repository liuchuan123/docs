title: "Get Started with PayMatrix in Minutes"
sidebarTitle: "Quickstart"

---

This guide walks you through everything you need to go from zero to a completed payment using the PayMatrix API. By the end, you'll have created an order, attached a payment to it, and confirmed the transaction succeeded — all in the sandbox environment so there's no risk to real funds.

<Tip>
  All requests in this guide use test API keys that begin with `pm_test_`. These keys target the PayMatrix sandbox environment, which behaves identically to production but never moves real money. Switch to a live key only when you're ready to go live.
</Tip>

<Steps>

  <Step title="Create your account">
    Head to [dashboard.paymatrix.com](https://dashboard.paymatrix.com) and sign up for a free account. During onboarding you'll provide basic business information — your business name, country, and the type of payments you plan to accept. Onboarding typically takes under two minutes.

    Once you complete onboarding, you'll land on the PayMatrix dashboard where you can manage keys, view transactions, and configure webhooks.
  </Step>

  <Step title="Get your API key">
    In the dashboard, navigate to **Settings → API Keys**. You'll see two key types listed:

    - **Test keys** (`pm_test_...`) — for development and integration testing against the sandbox
    - **Live keys** (`pm_live_...`) — for production requests that process real money

    Copy your test key. You'll include it in the `Authorization` header of every request you make to the PayMatrix API.
  </Step>

  <Step title="Create an order">
    An order represents the intent to collect payment for a specific amount. Create one by sending a `POST` request to `/v1/orders` with the amount in the smallest currency unit (e.g. cents for USD), the currency code, and an optional description.

    <CodeGroup>
      ```bash Request
      curl -X POST https://api.paymatrix.com/v1/orders \
        -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
        -H "Content-Type: application/json" \
        -d '{
          "amount": 5000,
          "currency": "USD",
          "description": "Order #1001",
          "metadata": { "customer_id": "cust_abc123" }
        }'
      ```

      ```json Response
      {
        "id": "ord_1A2B3C4D5E",
        "status": "pending",
        "amount": 5000,
        "currency": "USD",
        "description": "Order #1001",
        "created_at": "2024-01-15T10:30:00Z"
      }
      ```
    </CodeGroup>

    Save the `id` field from the response (e.g. `ord_1A2B3C4D5E`) — you'll reference it in the next step to attach a payment.
  </Step>

  <Step title="Create a payment">
    With your order created, submit a payment against it by posting to `/v1/payments`. Pass the `order_id` you received in the previous step, the payment method, and the card details. In the sandbox, use the test card number `4242424242424242` to simulate a successful charge.

    <CodeGroup>
      ```bash Request
      curl -X POST https://api.paymatrix.com/v1/payments \
        -H "Authorization: Bearer pm_test_xxxxxxxxxxxx" \
        -H "Content-Type: application/json" \
        -d '{
          "order_id": "ord_1A2B3C4D5E",
          "method": "card",
          "card": {
            "number": "4242424242424242",
            "exp_month": 12,
            "exp_year": 2027,
            "cvc": "123"
          }
        }'
      ```

      ```json Response
      {
        "id": "pay_9Z8Y7X6W5V",
        "order_id": "ord_1A2B3C4D5E",
        "status": "succeeded",
        "amount": 5000,
        "currency": "USD",
        "method": "card",
        "created_at": "2024-01-15T10:30:05Z"
      }
      ```
    </CodeGroup>

    A `status` of `succeeded` means the payment was authorized and captured. Your order is now paid.
  </Step>

  <Step title="Check payment status">
    At any time you can retrieve the current state of a payment by sending a `GET` request to `/v1/payments/{id}`, substituting the payment ID you received in the previous step.

    ```bash
    curl https://api.paymatrix.com/v1/payments/pay_9Z8Y7X6W5V \
      -H "Authorization: Bearer pm_test_xxxxxxxxxxxx"
    ```

    The response returns the same payment object you saw above, with the latest `status` value. Common statuses include `pending`, `processing`, `succeeded`, and `failed`.
  </Step>

  <Step title="Switch to live">
    When you're ready to accept real payments, return to **Settings → API Keys** in the dashboard and copy your live key (it begins with `pm_live_`). Replace your test key with the live key in your integration — no other code changes are required.

    <Warning>
      Live keys process real charges against real payment methods. Never use a live key in a development or staging environment, and never commit any API key to version control.
    </Warning>
  </Step>

</Steps>
