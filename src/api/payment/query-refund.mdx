title: "退款查询"
description: "根据平台退款交易号查询退款订单的当前状态与详细信息。"
openapi: "GET /api/merchant/order/refund/query/{refundId}"
---

根据**平台退款交易号（`refund_id`）**查询退款订单的当前状态。

## 接口信息

| 项目         | 说明                                                |
| ------------ | --------------------------------------------------- |
| 请求方式     | `GET`                                               |
| 路径         | `/api/merchant/order/refund/query/{refundId}`       |
| Content-Type | —（GET 请求无 Body）                                |
| 鉴权方式     | RSA 签名                                            |

## 路径参数

| 参数       | 类型   | 必填 | 说明                                             |
| ---------- | ------ | ---- | ------------------------------------------------ |
| `refundId` | String | 是   | 平台退款交易号（发起退款接口返回的 `refund_id`） |

<Note>
  查询使用的是**平台退款交易号 `refund_id`**，不是商户退款单号 `merchant_refund_no`。
</Note>

## 响应参数

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "refund_id": "26020100098765432",
    "order_id": "26020100012345678",
    "merchant_id": "10001",
    "merchant_refund_no": "REF-20260201-0001",
    "refund_amount": 50.00,
    "refund_currency": "USD",
    "status": 2,
    "reason": "用户申请退款",
    "remark": "商品有瑕疵"
  }
}
```

### 字段说明

| 字段                 | 类型       | 说明               |
| -------------------- | ---------- | ------------------ |
| `refund_id`          | String     | 平台退款交易号     |
| `order_id`           | String     | 关联的订单号       |
| `merchant_id`        | String     | 商户 ID            |
| `merchant_refund_no` | String     | 商户退款单号       |
| `refund_amount`      | BigDecimal | 退款金额           |
| `refund_currency`    | String     | 退款币种           |
| `status`             | Integer    | 退款交易状态       |
| `reason`             | String     | 退款原因           |
| `remark`             | String     | 退款备注           |

### 退款交易状态

| 状态码 | 状态   | 说明         |
| ------ | ------ | ------------ |
| 0      | 初始化 | 退款已创建   |
| 1      | 处理中 | 退款处理中   |
| 2      | 成功   | 退款已完成   |
| 5      | 已取消 | 退款已取消   |
| 6      | 失败   | 退款处理失败 |

## 使用场景

- 确认退款是否到账
- Webhook 未收到时的状态确认
- 后台对账核对退款记录
