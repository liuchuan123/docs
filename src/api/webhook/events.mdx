{{BRAND}} 通过 Webhook 推送订单、退款与争议事件。事件类型为 `.` 分隔的小写字符串。

## 支付事件

| 事件类型                     | 说明         | 可订阅 | 当前推送 |
| ---------------------------- | ------------ | ------ | -------- |
| `order.payment.succeeded`    | 支付成功     | ✔      | ✔        |
| `order.payment.failed`       | 支付失败     | ✔      | ✔        |
| `order.payment.created`      | 支付订单创建 | —      | 预留     |
| `order.payment.processing`   | 支付处理中   | —      | 预留     |
| `order.payment.cancelled`    | 支付取消     | —      | 预留     |

## 退款事件

| 事件类型                  | 说明         | 可订阅 | 当前推送 |
| ------------------------- | ------------ | ------ | -------- |
| `order.refund.succeeded`  | 退款成功     | ✔      | ✔        |
| `order.refund.failed`     | 退款失败     | ✔      | ✔        |
| `order.refund.created`    | 退款申请创建 | —      | 预留     |
| `order.refund.processing` | 退款处理中   | —      | 预留     |

## 争议事件

| 事件类型                     | 说明         | 可订阅 | 当前推送 |
| ---------------------------- | ------------ | ------ | -------- |
| `order.dispute.created`      | 争议创建     | ✔      | ✔        |
| `order.dispute.won`          | 商户胜诉     | ✔      | ✔        |
| `order.dispute.lost`         | 商户败诉     | ✔      | ✔        |
| `order.dispute.closed`       | 争议关闭     | ✔      | ✔        |
| `order.dispute.evidence_due` | 待提交证据   | —      | 预留     |

<Note>
  标记为「预留」的事件已在事件目录中定义，但当前版本尚未推送。请以「当前推送」列为准进行订阅与联调。结算（`settlement.*`）与账户（`account.*`）类事件同样属于预留，暂未推送。
</Note>

## Payload 通用结构

所有事件的 HTTP Body 统一为如下信封结构，`data` 内容随事件类型变化：

```json
{
  "event_id": "evt_order.payment.succeeded_10001_26020100012345678",
  "event_type": "order.payment.succeeded",
  "timestamp": 1749081600,
  "data": { }
}
```

| 字段         | 类型   | 说明                                     |
| ------------ | ------ | ---------------------------------------- |
| `event_id`   | String | 事件唯一 ID，用于幂等去重                |
| `event_type` | String | 事件类型                                 |
| `timestamp`  | Long   | 事件发生时间（Unix **秒**）              |
| `data`       | Object | 事件负载，字段随事件类型变化（snake_case）|

<Note>
  信封的 `timestamp` 为 Unix 秒；`data` 内的时间字段（如 `success_time`、`refund_time`、`respond_by` 等）统一为 `YYYY-MM-DD HH:mm:ss` 格式，与[支付查询 API](/api/payment/query) 一致。**所有时间均为 UTC。**
</Note>

## 订阅与匹配

订阅时 `events` 支持两种写法：

- **精确匹配**：如 `order.payment.succeeded`
- **前缀通配**：以 `.*` 结尾，如 `order.payment.*`、`order.*`（匹配该前缀下的所有事件）

| 场景         | 建议订阅                                                        |
| ------------ | -------------------------------------------------------------- |
| 订单状态同步 | `order.payment.succeeded`、`order.payment.failed`              |
| 退款处理     | `order.refund.succeeded`、`order.refund.failed`                |
| 争议处理     | `order.dispute.created`、`order.dispute.won`、`order.dispute.lost`、`order.dispute.closed` |
| 全面监控     | `order.*`                                                     |
