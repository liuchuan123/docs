{{BRAND}} 按平台配置、商户签约能力和支付渠道可用性支持交易与结算币种。请以商户门户展示的可用币种和签约协议为准。

## 交易币种

请求中的 `currency_code` 使用 [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217) 三位币种代码。代码符合 ISO 标准不代表该币种已为你的商户或当前渠道开通；未配置的币种会被接口拒绝。

| 币种代码 | 币种名称 | 零头单位 |
| -------- | -------- | -------- |
| `USD` | 美元 | 美分（0.01） |
| `EUR` | 欧元 | 欧分（0.01） |
| `GBP` | 英镑 | 便士（0.01） |
| `CNY` | 人民币 | 分（0.01） |
| `JPY` | 日元 | 无（最小单位 1） |
| `HKD` | 港币 | 毫（0.01） |
| `SGD` | 新加坡元 | 分（0.01） |
| `AUD` | 澳大利亚元 | 分（0.01） |
| `CAD` | 加拿大元 | 分（0.01） |
| `CHF` | 瑞士法郎 | 生丁（0.01） |
| `KRW` | 韩元 | 无（最小单位 1） |
| ... | 其他已签约且已配置的币种 | 以门户和协议为准 |

<Note>
  具体可用的币种取决于商户配置、所选支付渠道和签约协议。日元（JPY）、韩元（KRW）等币种无小数位，最小单位为 1。
</Note>

## 结算币种

结算币种指结算到商户账户余额时使用的币种。是否与交易币种相同、可选币种及结算安排均以商户签约协议和实际配置为准。

## 币种转换

如果交易币种与结算币种不同，适用的换汇规则、汇率口径和费用以签约协议与结算单明细为准。请勿假设所有交易都会自动换汇。

## 金额格式注意事项

- 所有金额使用数值类型，JSON 中直接传数字
- 小数位必须符合平台为该币种配置的精度；不要假设所有币种均为两位小数
- 正确示例：`100.00` 表示 100 美元
- 不要使用科学计数法：错误 `1.0E2`，正确 `100.00`
- 不要使用千分位分隔符：错误 `1,000.00`，正确 `1000.00`
