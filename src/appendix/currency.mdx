{{BRAND}} 支持全球主流币种，覆盖国际支付场景。

## 交易币种

平台支持 [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217) 标准的所有活跃币种，包括但不限于：

| 币种代码 | 币种名称 | 零头单位 |
| -------- | -------- | -------- |
| `USD` | 美元 | 美分（0.01） |
| `EUR` | 欧元 | 欧分（0.01） |
| `GBP` | 英镑 | 便士（0.01） |
| `CNY` | 人民币 | 分（0.01） |
| `JPY` | 日元 | 无（最小单位 1） |
| `HKD` | 港币 | 毫（0.01） |
| `SGD` | 新加坡元 | 分（0.01） |
| `AUD` | 澳大利亚元 | 分（0.01） |
| `CAD` | 加拿大元 | 分（0.01） |
| `CHF` | 瑞士法郎 | 生丁（0.01） |
| `KRW` | 韩元 | 无（最小单位 1） |
| ... | 更多币种持续支持中 | |

<Note>
  具体可用的币种取决于所选支付渠道。日元（JPY）、韩元（KRW）等币种无小数位，最小单位为 1。
</Note>

## 结算币种

结算币种指结算到商户账户余额时使用的币种，默认为交易币种。

支持多种结算币种，包括 `USD`、`EUR`、`CNY`、`HKD`、`GBP`、`SGD` 等。具体以商户签约协议为准。

## 币种转换

如果交易币种与结算币种不同，平台在结算时会自动进行币种转换：

- 汇率以结算日当天银行间汇率为准
- 转换费用在手续费明细中体现

## 金额格式注意事项

- 所有金额使用数值类型，JSON 中直接传数字
- 大多数币种最多两位小数（日元、韩元等除外）
- 正确示例：`100.00` 表示 100 美元
- 不要使用科学计数法：错误 `1.0E2`，正确 `100.00`
- 不要使用千分位分隔符：错误 `1,000.00`，正确 `1000.00`
