在正式对外提供服务之前，建议先在测试环境完成联调，验证签名、下单、收银台、Webhook 与退款的完整闭环。

<Note>
  测试所需的**测试商户、密钥与地址**由平台开通并提供。联调期间不涉及真实资金流动。请联系客户经理获取测试凭据。
</Note>

## 联调建议

<Steps>
  <Step title="获取测试凭据">
    从平台获取测试商户 ID、上传测试用 RSA 公钥，并确认联调地址。
  </Step>
  <Step title="打通签名">
    先用 [创建支付](/api/payment/create) 接口验证 RSA 签名是否通过，排除签名原文与请求头问题。
  </Step>
  <Step title="跑通支付闭环">
    创建订单 → 跳转 `pay_url` 收银台 → 完成支付 → 接收 Webhook → 查询核对。
  </Step>
  <Step title="验证退款">
    对已支付订单发起全额/部分退款，确认退款 Webhook 与退款查询结果。
  </Step>
  <Step title="验证异常与幂等">
    验证重复下单、超时补单、Webhook 幂等（同一 `event_id` 去重）等边界场景。
  </Step>
</Steps>

## 需要覆盖的场景

### 支付

- 支付成功
- 支付失败
- 支付处理中 → 最终成功/失败（异步确认）
- 3DS 认证流程
- 超时未支付关闭

### 退款

- 全额退款成功
- 部分退款成功（多次部分退款）
- 退款失败

### Webhook

- `order.payment.succeeded` / `order.payment.failed`
- `order.refund.succeeded` / `order.refund.failed`
- 争议事件 `order.dispute.*`

## 联调调试工具

在商户门户，你可以使用以下工具排查问题：

- **Webhook 测试发送**：手动触发指定事件类型的测试推送
- **Webhook 投递记录**：查看最近投递历史与响应，并对失败投递重试

## 上线检查清单

- [ ] 签名联调通过，签名原文与 4 个请求头正确
- [ ] 支付、退款、Webhook、查询闭环全部验证通过
- [ ] Webhook 已做验签与 `event_id` 幂等处理
- [ ] 生产环境 RSA 公钥已上传
- [ ] IP 白名单已配置（如启用）
- [ ] 已用小金额真实交易验证生产链路
- [ ] 关键异常有日志记录与告警
