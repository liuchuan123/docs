## 支付订单列表

按时间范围、订单状态、支付方式等条件筛选查看：

| 字段 | 说明 |
| ---- | ---- |
| 订单 ID | 平台生成的唯一订单号 |
| 商户订单号 | 商户创建支付时传入的交易 ID |
| 金额 | 订单金额与币种 |
| 支付方式 | 用户使用的支付方式 |
| 状态 | 订单当前状态 |
| 创建时间 | 订单创建时间 |
| 支付成功时间 | 用户实际完成支付的时间 |

列表支持导出为 Excel 文件，便于对账。除订单列表外，你也可以按交易/支付流水进行查询。

## 交易详情

点击订单可查看详情，包含：

- 订单信息（ID、金额、币种、状态、时间）
- 商品信息（商品列表、单价、数量）
- 客户信息（姓名、地址）
- 收货信息
- 支付流水（每次支付尝试的详情）
- 退款记录（如有关联退款）
- 费用明细（平台手续费）

## 退款管理

退款在**交易详情页**发起，支持部分退款。退款列表展示所有退款记录：

| 字段 | 说明 |
| ---- | ---- |
| 退款 ID | 平台退款单号 |
| 商户退款号 | 商户传入的退款号 |
| 原订单 ID | 关联的支付订单 |
| 退款金额 | 退款金额与币种 |
| 状态 | 退款处理状态 |
| 退款时间 | 退款发起时间 |
| 退款原因 | 商户填写的退款原因 |

### 发起退款

在交易详情页面点击「退款」按钮：

<Steps>
  <Step title="输入退款金额">
    支持部分退款，金额不超过可退余额。
  </Step>
  <Step title="填写退款原因">
    填写退款原因与备注。
  </Step>
  <Step title="确认提交">
    确认后提交退款申请。
  </Step>
</Steps>

退款结果将通过 Webhook 通知，详见 [Webhook 概览](/api/webhook/overview)。

## 争议跟踪

争议（Dispute / Chargeback）列表是**只读**的跟踪视图，展示所有争议订单的状态、争议金额与处理截止时间，供你了解争议进展。

<Note>
  争议列表本身不支持在门户内提交抗辩材料。若你需要就风控处置或争议主动提出救济，请使用单独的「风控申诉（Risk Appeal）」功能提交申诉。
</Note>
