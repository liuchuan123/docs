---
title: "账务体系"
description: "PMP Payment 账务体系架构说明"
---


PMP Payment 采用四层账务体系，确保每一笔资金变动可追溯、可对账。

## 账务层次

```
交易层 (Transaction)
    ↓
渠道账本 (Channel Ledger)
    ↓
平台总账 (Platform General Ledger)
    ↓
商户账本 (Merchant Ledger)
    ↓
余额快照 (Balance Snapshot)
```

### 交易层

记录每一笔支付、退款、争议等交易的原始数据，是整个账务体系的基础。

关键字段：
- 交易金额、币种
- 渠道手续费
- 平台手续费
- 交易时间
- 交易状态

### 渠道账本

按支付渠道（Stripe、Antom 等）汇总交易数据，用于与渠道方对账。

### 平台总账

平台级的资金汇总，反映平台整体的资金流入流出。

采用复式记账：
- **借方（Debit）**：资金流入
- **贷方（Credit）**：资金流出

每笔交易在总账中产生对应的借贷分录。

### 商户账本

每个商户拥有独立的账本，记录商户维度的资金变动。

## 商户账户类型

| 账户     | 类型 | 说明                             |
| -------- | ---- | -------------------------------- |
| 可用账户 | 资产 | 已完成结算、可提现的余额         |
| 保证金账户 | 资产 | 预留风险保证金                   |
| 冻结账户 | 资产 | 因争议等被冻结的金额             |
| 待结算账户 | 资产 | 已支付但未到结算日的金额         |

## 余额快照

系统定期生成余额快照，用于：
- 快速查询当前余额（无需实时汇总所有流水）
- 对账校验（与流水汇总对比，发现差异）
- 结算时确定结算金额

## 资金流转示意

### 支付场景

```
消费者支付 $100（手续费：渠道 $2，平台 $1）
─────────────────────────────────────────────
渠道到账：         $100
渠道手续费：       -$2   (渠道账本)
平台手续费：       -$1   (平台总账)
商户待结算：       +$97  (商户待结算账户)

结算日：
商户待结算：       -$97
商户可用：         +$97  (商户可用账户)

提现：
商户可用：         -$97
银行到账：         +$97
```

### 退款场景

```
退款 $50（手续费按比例退还或不退，视渠道政策）
─────────────────────────────────────────────
商户可用/待结算：   -$50
渠道退款：         +$50 → 消费者
```

## 对账

平台提供多维度对账能力：

- **交易对账**：商户交易记录与平台交易记录核对
- **结算对账**：商户待结算金额与结算单核对
- **渠道对账**：平台记录与渠道方账单核对

建议商户每日进行交易对账，确保系统记录与平台一致。
